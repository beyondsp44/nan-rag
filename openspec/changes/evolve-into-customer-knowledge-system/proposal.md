# Proposal: Evolve NAN Aesthetics RAG into a customer knowledge system

## Summary

將目前以單次 QA 檢索為主的醫美 RAG 展示頁，逐步擴展為更接近「專有知識庫客服系統」的產品型專案，讓系統不只可展示，也能支援更廣的顧客提問、知識管理與客服回應場景。

## Why

目前專案已具備以下基礎：

- 可公開展示的 Streamlit 介面
- 穩定的 Supabase 檢索流程
- 醫美情境化的品牌型 UI
- 可持續擴充的 QA 資料來源

但目前仍偏向「相似 QA 命中展示」。若要更接近專有知識庫客服系統，還需要把知識更新、分類檢索、回答重組與後台管理一起納入規劃。

## Candidate Enhancements

### 1. 多分類知識庫檢索

- 依療程、術前、術後、價格、恢復期、禁忌、適用族群分類
- 讓查詢前可先選分類，縮小檢索範圍
- 為未來多資料表或多知識庫來源做準備

### 2. 回答重組與客服語氣輸出

- 不只回傳最接近的單筆 QA
- 可綜合前幾筆命中內容，整理成更像顧問回覆的答案
- 保留原始來源依據，避免產生難以追蹤的內容

### 3. 知識庫更新流程標準化

- 定義 QA 新增、修改、刪除後的更新流程
- 明確規範 embedding 重建與回寫步驟
- 降低資料更新後查詢結果不同步的風險

### 4. 管理端能力

- QA 匯入介面
- 查詢紀錄檢視
- 命中品質檢查
- 常見問題統計與空白查詢分析

### 5. 展示頁升級為客服入口

- 新增熱門問題分類入口
- 新增療程導覽與術前術後常見問答
- 讓畫面更接近真實客服知識平台，而不只是單一搜尋框

## Recommendation

建議分三階段進行：

1. 先做知識庫更新流程標準化與分類檢索
2. 再做回答重組與客服語氣輸出
3. 最後再補管理端與進階展示入口
