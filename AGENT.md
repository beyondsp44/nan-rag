# AGENT.md

## 專案目標
請協助建立一個教學用的 RAG 客服前端雛形，使用 **Python 3.12** 與 **Streamlit**，讓使用者可以在網頁上輸入問題，並顯示檢索或回答結果。

本專案對象是 **零程式基礎的新手學生**，因此請遵守以下原則：

- 程式碼結構簡單
- 命名清楚
- 加入適量中文註解
- 優先確保可執行，再考慮美化
- 不要過度設計，不需要複雜架構
- 先完成本地可執行版本

---

## 你要完成的任務

### 1. 建立 Python 3.12 虛擬環境
請在專案中建立適合本地開發的 Python 3.12 虛擬環境設定，並提供必要的安裝步驟。

需求：
- 使用 **Python 3.12**
- 建立虛擬環境
- 安裝必要套件
- 產生 `requirements.txt`

必要套件至少包含：
- `streamlit`
- `supabase`
- `sentence-transformers`
- `python-dotenv`

如果有需要，可加入：
- `pandas`

請同時在 `README.md` 中寫出啟動方式。

---

### 2. 製作 Streamlit 客戶端頁面
請建立一個簡潔的 Streamlit 頁面，提供使用者輸入問題。

基本需求：
- 頁面標題
- 問題輸入框
- 提交按鈕
- 顯示查詢結果區塊

互動流程：
1. 使用者輸入問題
2. 按下按鈕後進行查詢
3. 顯示回答內容
4. 若查無結果，顯示友善提示訊息

請先預留函式介面，例如：
- `embed_query(query: str)`
- `search_qa(query: str)`

如果 Supabase 查詢邏輯尚未完整，也請先保留可替換的函式骨架，不要卡住整體頁面。

---

### 3. 頁面稍微美化
請在 **不複雜化** 的前提下，將 Streamlit 頁面做基本美化。

美化方向：
- 版面置中或留白適當
- 標題清楚
- 使用 `st.container()`、`st.columns()`、`st.markdown()` 做簡單排版
- 回答結果區塊有視覺區隔
- 整體風格乾淨、簡單、教學友善

請避免：
- 過多動畫
- 過度花俏配色
- 複雜 CSS
- 不必要的套件

---

## 專案建議結構
請盡量整理成以下結構：

```text
project/
├─ app.py
├─ rag_search.py
├─ .env.example
├─ requirements.txt
└─ README.md
```

## 檔案要求
### app.py

負責：
```
Streamlit 畫面
接收使用者輸入
呼叫查詢函式
顯示結果
```

### rag_search.py

負責：
```
載入 embedding 模型
連接 Supabase
提供查詢函式
保持程式簡單清楚
```

### .env.example

請包含以下變數名稱：
```
SUPABASE_URL=your_supabase_url
SUPABASE_KEY=your_supabase_key
```

### requirements.txt

請列出可直接安裝的套件版本，避免太新或不穩定版本。

### README.md

請寫清楚：
```
如何建立 Python 3.12 虛擬環境
如何安裝套件
如何設定 .env
如何啟動 Streamlit
```

## 程式撰寫原則

請遵守以下規則：
1. 先求可執行
   1. 不要先做複雜架構
   2. 先完成最小可行版本
2. 保留教學可讀性
   1. 加入中文註解
   2. 讓新手能看懂
3. 模組化但不要過度
   1. app.py 負責前端
   2. rag_search.py 負責查詢邏輯
4. 錯誤處理要友善
   1. 若 .env 未設定，請提示使用者
   2. 若查詢失敗，請顯示易懂訊息
5. 不要自動加入部署設定
   1.  這一版先以本地執行為主
   2.  不需要先加入 Docker、CI/CD、雲端部署腳本

## 介面文案建議

請使用繁體中文介面，例如：
 * 頁面標題：RAG 客服查詢系統
 * 輸入提示：請輸入您的問題
 * 按鈕文字：開始查詢
 * 結果標題：查詢結果
 * 無結果提示：查無相符資料，請換個方式提問。

## 回傳結果格式

若查詢成功，請優先顯示：
1. 最相關答案
2. 相似問題（可選）
3. 相似度分數（可選，若實作容易）

若目前後端尚未完成，可先用假資料 mock 畫面，但請在程式註解中標明可替換位置。

請完成一個可以本地執行的最小專案，讓使用者能透過 Streamlit 輸入問題，並看到查詢結果。
